# RenLern Hyper-V VM Tools - Quick Reference
## 3 Tools für komplettes Backup & Restore Management

---

## 🚀 **QUICK START (3 Befehle)**

### 1️⃣ Backup JETZT erstellen
```powershell
.\export_vm.ps1 -VMName "RenLern-Debian" -ExportPath "C:\Hyper-V\Exports"
```
✓ VM wird automatisch heruntergefahren  
✓ Snapshot erstellt für Konsistenz  
✓ Alle Einstellungen gespeichert (JSON)  
✓ Größe: ~30-50 GB je nach Daten  

---

### 2️⃣ Aus Backup wiederherstellen
```powershell
.\import_vm.ps1 -VMName "RenLern-Debian-Restored" -ImportPath "C:\Hyper-V\Exports" -StartVM
```
✓ Findet neustes Backup automatisch  
✓ Stellt CPU/RAM/Netzwerk wieder her  
✓ Startet VM automatisch  
✓ Zeigt neue VM-IP  

---

### 3️⃣ Interaktives Management-Menü
```powershell
.\vm_manager.ps1 -VMName "RenLern-Debian" -BackupPath "C:\Hyper-V\Backups"
```
Wähle:
- [1] Backup jetzt
- [2] Wiederherstellen  
- [3] VM klonen
- [4] Automatisierung
- [5] Speicher freigeben
- [6] Info anzeigen

## 🎯 **Häufige Szenarien**

### Scenario 1: Tägliches Backup (empfohlen)
```powershell
# Vor dem Shutdown
.\vm_manager.ps1 -VMName "RenLern-Debian" -Action backup

# Ergebnis: RenLern_Backup_YYYYMMDD_HHMMSS
```

### Scenario 2: Schnelle Wiederherstellung nach Fehler  
```powershell
# Gib alte Backups an
.\vm_manager.ps1 -VMName "RenLern-Debian" -Action restore

# Wähle Backup aus Liste
# Neue VM wird erstellt & gestartet
```

### Scenario 3: Parallel-System für Testing
```powershell
# Klone für Test-Umgebung
.\vm_manager.ps1 -VMName "RenLern-Debian" -Action clone

# Enters: "RenLern-Test"
# Beide VMs laufen parallel mit unterschiedlichen IPs
```

### Scenario 4: Automatische nächtliche Backups
```powershell
# Im Menü Option [4] wählen
.\vm_manager.ps1 -VMName "RenLern-Debian" -BackupPath "C:\Hyper-V\Backups"

# Wähle: [1] Täglich um 02:00
# Dann: Windows Task Scheduler übernimmt
```

### Scenario 5: Alte Backups löschen (Speicher freigeben)
```powershell
.\vm_manager.ps1 -VMName "RenLern-Debian" -BackupPath "C:\Hyper-V\Backups" -Action cleanup

# Wähle: [3] Behalte nur letzte 5
# Löscht automatisch alte Backups
```

## 🔧 **Parameter Übersicht**

### export_vm.ps1
```powershell
-VMName        # QB: "RenLern-Debian"
-ExportPath    # Default: "C:\Hyper-V\Exports"
-Compress      # Optional: ZIP-Datei erstellen
-NoCleanup     # Optional: Originalordner behalten
```

### import_vm.ps1
```powershell
-VMName             # Default: "RenLern-Debian-Restored"
-ImportPath         # Default: "C:\Hyper-V\Exports"
-StartVM            # Optional: VM nach Import starten
-SkipConfiguration  # Optional: Nur VM kopieren nicht Einstellungen
```

### vm_manager.ps1
```powershell
-VMName       # Default: "RenLern-Debian"
-BackupPath   # Default: "C:\Hyper-V\Backups"
-Action       # "menu" | "backup" | "restore" | "clone"
```

## 💡 **Pro-Tipps**

### Tip 1: Backup mit Komprimierung (spart ~20 GB)
```powershell
.\export_vm.ps1 -VMName "RenLern-Debian" -ExportPath "C:\Hyper-V\Exports" -Compress
# Resultat: .ZIP Datei statt Ordner
```

### Tip 2: Mehrere Backups automatisch
```powershell
# Setup im vm_manager -> Option 4
# Dann läuft jeden Tag um 02:00 Uhr automatisch
```

### Tip 3: Backup auf NAS kopieren
```powershell
robocopy C:\Hyper-V\Exports \\NAS\RenLern_Backups /MIR /Z
```

### Tip 4: Backup-Größe prüfen
```powershell
$backups = Get-ChildItem C:\Hyper-V\Exports -Directory
foreach ($b in $backups) {
    $size = (Get-ChildItem $b -Recurse | Measure-Object -Sum -Property Length).Sum
    Write-Host "$($b.Name): $([math]::Round($size/1GB,2))GB"
}
```

### Tip 5: Schnelle Test-VM
```powershell
# Ohne Konfiguration = schneller
.\import_vm.ps1 -VMName "RenLern-QuickTest" -ImportPath "..." -SkipConfiguration
```

```
1. VM läuft in Produktion
          ↓
2. [Option 1] Backup erstellen
          ↓
3. RenLern_Export_YYYYMMDD_HHMMSS/ gespeichert
          ↓
4. VM getestet bei Bedarf
          ↓

          ↓
```

## 📞 **Hilfe & Fehlerbehebung**

### "VM hinzufügen fehlgeschlagen"
→ Siehe Seite 2 in **VM_MANAGEMENT.md** / Troubleshooting

### "Speicherplatz voll"
→ Option [5] im vm_manager zum Cleanup
→ Oder: Alte Backups auf externe Festplatte verschieben

### "Import dauert lange"
→ Mit `-SkipConfiguration` schneller
→ Oder: Nur VHDX (ohne RAM/CPU Restore)
